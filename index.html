buppy file storage

What does it do?

Encrypted and signed file storage

The command line client encrypts, deduplicates and signs files/folders before sending them to a remote server via ssh. You can then retrieve your data via the command line, or a browser interface.

It is easy to setup and requires only ssh

Copy the 'bpy' binary to your client machine and to a server with ssh enabled, and you are ready to go.

Has an open source implementation

The command line tool 'bpy' is standalone, free and open source. Don't trust the client to keep your data safe? Browse the code!

Download and Setup - (PreAlpha)

Buppy is easy to install. Download a single binary with zero run-time dependencies and run it from the command line.

Step 1: Download bpy for your platform and put the bpy executeable in your path

On Windows you can run the msi installer, on Linux, BSD or OSX you can extract the zip or tarball and add the 'bpy' binary to your $PATH environment variable

Step 2: Create a key file

Keep a copy of this file in a safe location such as a flash drive, or google account. If you lose it your data is lost FOREVER. When accessing your data you need both the key file, and the encrypted data files

$ bpy new-key -f ~/.bpy/bpy.key

Step 3: Set the BPY_REMOTE_CMD environment variable

This environment variable should be a command to run a 'bpy remote' process remotely. Typically this command will be an ssh command such as:

$ export BPY_REMOTE_CMD="ssh $SERVER bpy remote /home/$USER/bpy_drive/"

Step 4: Run it!

$ bpy put files.txt
$ bpy ls
